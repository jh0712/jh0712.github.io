RESTful API 設計主要原則

核心原則

1. 使用 HTTP 方法（HTTP Verbs）

  • GET: 獲取資源
  • POST: 創建新資源
  • PUT: 完整更新資源
  • PATCH: 部分更新資源
  • DELETE: 刪除資源

2. 使用名詞而非動詞

5. 使用適當的 HTTP 狀態碼

  • 200 OK: 請求成功
  • 201 Created: 資源創建成功
  • 204 No Content: 成功但無返回內容
  • 400 Bad Request: 請求錯誤
  • 401 Unauthorized: 未授權
  • 403 Forbidden: 禁止訪問
  • 404 Not Found: 資源不存在
  • 500 Internal Server Error: 伺服器錯誤
